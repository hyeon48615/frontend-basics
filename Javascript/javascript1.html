자바스크립트 작성

  • 1. 웹문서 안에 작성
    • <script> 태그와 </script> 태그 안에 작성
    • 웹 문서의 어느 곳에든 위치할 수 있지만 주로 참조하는 개체의 뒤쪽에 작성하며, </body> 태그 앞에 작성한다.
    • 자바스크립트 소스가 작성된 위치에서 실행됨
  • 2. 외부 스크립트 파일에 작성
    • 자바스크립트 소스를 별도의 파일(.js) 로 만들어 참조
    • 기본형 <script src="스크립트 파일경로"></script>
  • 개발 표준 가이드
    • 자바스크립트에서 개발 가이드가 더 필요한 이유는 디버깅이 불편하다.
    • 오픈소스로 개발하거나, 외부의 다른 사람과 공유 시 가독성과 관리가 편리하다.
    • 팀 프로젝트 진행 시, 정규화된 코딩 규칙이 필요하다.
    • 유지보수가 용이하고, 비용 절감 효과를 가져올 수 있다.
    • 구글 스타일 가이드나, 혹은 다른 개발사의 가이드를 참조한다.
    • 구글 스타일 가이드 : https://google.github.io/styleguide/jsguide.html
    • 에어비앤비 스타일가이드 : https://github.com/tipjs/javascript-style-guide
  • 개발 규칙
    • 들여쓰기를 맞춘다. -> 사람을 위한 부분
    • 세미콜론으로 구분
    • 구문 사이에 공백 등을 넣어서 가독성을 높인다.
    • 주석을 활용한다.
    • 언어별 시스템 규칙에 맞추어서 코딩을 진행한다.
    • 개발 가이드와 현업이 다른 경우 현업이 우선시되어야 한다.
    • Javascript는 디버깅이 불편하므로, console 로그를 최대한 활용한다.
