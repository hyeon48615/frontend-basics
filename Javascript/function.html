함수

  • 기본 구문
    • 동작해야 할 목적대로 명령, 구문을 묶어 놓은 것
    • 각 명령의 시작, 끝을 명확하게 구분할 수 있음
    • 묶는 기능에 이름을 붙여서 같은 이름으로 명령을 실행할 수 있음
    • 의미있는 함수가 자바스크립트에 내장되어 있음
    • 유지보수 용이
    • 함수선언 : 어떤 명령을 처리할 지 미리 알려주는 것
    • 기본형
      함수 선언 : function 함수명() { 실행구문; }
      function 함수명() { 실행구문; return; }
    • 함수 호출 : 함수명(), 함수명(변수)
  • 스코프
    • 변수가 적용되는 범위
    • 스코프에 따라서 지역변수(로컬), 전역변수(글로벌)로 나뉨
    • 함수스코프
      • 함수에서 정의한 블록문 스코프 유효 범위로 인정
      • 함수 내부는 지역스코프, 함수 외부는 전역스코프
    • 블록스코프
      • {} 로 구성된 블록문 기준으로 스코프의 유효 범위를 구분
      • let과 const 키워드로 선언한 변수
    • 지역스코프
      • 함수 내부에 선언한 변수 a 는 함수 내부에서만 참조 가능
      • 지역변수 : 지역스코프에 선언한 변수
    • 전역스코프
      • 스코프와 상관없이 모두 참조 가능
      • 전역변수 : 전역 스코프에 선언한 변수
    • 참조 우선순위
      • let, const 키워드는 같은 스코프 영역에서 중복 선언이 불가능
      • 전역스코프와 지역스코프에 같은 식별자를 가지는 참조 대상이 있다면,
        • 먼저 같은 스코프의 식별자를 참조
        • 같은 스코프 내에 식별자가 없을 경우 전역스코프를 찾음
  • 전역변수
    • 스크립트 소스 전체에서 사용
    • 함수 밖에서 선언하거나 함수 안에서 var 없이 선언
  • 지역변수
    • 함수 안에서 선언하고, 함수 안에서만 사용
    • var 로 변수 이름을 지정
  • var 변수와 호이스팅
    • 변수를 뒤에서 선언하지만, 마치 앞에서 미리 선언한 것처럼 인식함
    • 함수 실행문을 앞에 두고 선언 부분을 뒤에 두더라도 앞으로 끌어올려서 인식
    • 코드를 선언과 할당으로 나누었을 때, 선언부를 스코프 최상위 끌어올리는 것
    • 호이스팅의 대상
      • 함수선언문
      • var 키워드를 사용한 함수 표현식
      • 화살표 함수 방식
    • let, const 키워드로 선언하면 호이스팅이 되지 않음
    • 재선언과 재할당이 가능
      • 재선언 : 이미 선언한 변수를 다시 선언할 수 있음
      • 재할당 : 같은 변수에 달느 값을 할당하는 것
  • let을 사용한 변수의 특징
    • 블록 변수 - 블록({}) 안에서만 사용 가능
    • 재할당은 가능하지만 재선언은 할 수 없음
    • 호이스팅이 없음
    • 스코프의 범위가 다름
    • 변수명 중복이 불가능
  • const를 사용한 변수의 특징
    • 상수 - 변하지 않는 값을 선언할 때 사용
    • 재선언, 재할당 불가
    • const t = 값 할당; (선언과 동시에 값 할당)
    • const c = 값1 -> 값2 할당 불가
  • 자바스크립트 변수 사용 가이드
    • 전역 변수는 최소한으로 사용
    • var 변수는 함수의 시작 부분에서 선언 -> 가급적 사용 금지
    • for 문에서 카운터 변수는 let 변수로 선언
    • ES6 사용시 let 을 사용하는 것이 사용성이 좋음
  • 전역스코프가 오염됨 - 한 번만 사용할 함수인데, 식별자를 사용할 수 없게 되었을 때
  • 즉시 실행 함수
    • 함수를 정의하면서 동시에 실행까지 하는 함수
    • 한 번 실행되고 나면 메모리에 데이터가 남아 있지 않음
    • 해당 식별자를 한 번도 사용하지 않은 것처럼 인식
    • 기본형식 (function(){ 실행구문; })();
    • 기본형식 (function(매개변수){ 실행구문; }(인수));
  • 익명 함수
    • 함수 이름이 없는 함수
    • 함수 자체가 식이므로 함수를 변수에 할당할 수도 있고, 다른 함수의 매개변수로 사용할 수도 있음
    • 기본형 변수명 = function(변수1,,,) { 실행구문; }
  • 화살표 함수
    • ES6 이후 사용하는 표기법 : =>
    • 익명함수에서만 사용할 수 있음
    • 기본형 (매개변수) => { 함수 내용; };
    • 변수 = function() { 함수내용; return 리턴값; };
    • 변수 = ()=> { 함수내용; return 리턴값; };
    • 변수 = ()=> 단일문; (return 생략. 단일문의 결과가 바로 변수에 할당)
  • 이벤트
    • 웹 브라우저나, 사용자가 행하는 동작
    • 웹 문서 영역에서 이루어지는 동작만을 가리킴
    • 주로 마우스나, 키보드를 사용할 때, 웹 문서를 불러올 때, 폼에 내용을 입력할 때 등 발생
    • 문서 관련 이벤트
      • abort : 문서가 완전히 로딩되기 전에 불러오기를 멈췃을 때 발생
      • error : 문서가 정확히 로딩되지 않았을 때 발생
      • load : 문서 로딩이 끝나면 발생
      • resize : 문서 화면 크기가 바뀌었을 때 발생
      • scroll : 문서 화면이 스크롤 되었을 때 발생
      • unload : 문서에서 벗어날 때 발생
    • 마우스 이벤트
      • click : HTML 요소를 선택했을 때 발생
      • dbclick : HTML 요소를 더블클릭했을 때 발생
      • mousedown : HTML 요소에서 마우스 버튼을 눌렀을 때 발생
      • mouseup : HTML 요소 위에서 마우스 버튼을 클릭했다가 손을 뗄 때 발생
      • mouseover : HTML 요소 위에 마우스 포인터가 위치 했을 때 발생
      • mouseout : HTML 요소 위에서 마우스 포인터가 외부로 벗어날 때 발생
    • 키보드 이벤트
      • keydown : 키보드의 키를 누르는 순간 발생
      • keypress : 키보드의 키를 누르는 순간부터 손을 떼기 전까지 발생
      • keyup : 키보드의 키에서 손을 떼는 순간 발생
    • 폼 이벤트
      • blur : 폼 요소가 포커스를 잃었을 때
      • change : 목록이나, 체크박스, 레디오 등의 상태가 변경됐을 때, 주로 input, select, textarea 에서 사용
      • focus : 폼 요소에 포커스가 놓였을 때, label, select, textarea, button
      • reset : 폼이 리셋되었을 때
      • submit : submit 버튼을 클릭하거나, 폼의 서브밋이 발생했을 때
  • 이벤트 처리
    • 이벤트 처리기(이벤트 핸들러, event handler)
    • 이벤트가 발생했을 때, 처리하는 함수
    • 이벤트가 발생한 HML 태그에 직접 연결하여 사용
    • 기본형 <태그명 on이벤트명="처리할 함수명">
    • 이벤트 핸들러를 별도로 등록 - addEventListener 를 이용
